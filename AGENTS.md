# LiveTracker - Agent Instructions

## Workflows

| Action                  | Command                                          |
|-------------------------|--------------------------------------------------|
| Full build + test       | `./gradlew build`                                |
| Dev server              | `./gradlew -p backend quarkusDev`                 |
| Single module build     | `./gradlew :<module>:build`                       |
| JVM Docker image        | `./gradlew build` (then use `backend/src/main/docker/Dockerfile.jvm`) |
| Native Docker image     | `./gradlew build -Dquarkus.package.type=native`    |

## Architecture

Three Gradle modules. Kotlin 1.6.21, Quarkus 2.11.1.

* **`core/`** — Kotlin Multiplatform (JVM + JS). Shared domain models: `LiveEvent`, `Comment`, `Contest`, `Customer`, `Sku`. Tests in `commonTest`.
* **`backend/`** — Quarkus JVM app. REST/WebSocket APIs, Facebook Graph API client, services. Depends on `:core` and bundles `:frontend` output.
* **`frontend/`** — Kotlin/JS (React + MUI). Bundled by webpack and copied into backend's `META-INF/resources` during `processResources`.

## Build Order Matters

`backend:processResources` depends on `:frontend:browserWebpack`. Running `:backend:build` alone will fail if the frontend hasn't been built. Always use root `./gradlew build` or build frontend first.

## Mock Server

A local mock server (`MockServer.kt`) replaces the Facebook Graph API in dev/test profiles. It listens on port 8080 and serves `localhost:8080/mock`. No real Facebook credentials needed for local development.

## Testing

* Backend tests: JUnit5 + QuarkusTest + RestAssured in `backend/src/test/kotlin/`
* Core tests: Kotlin test framework in `core/src/commonTest/`
* Test profile reads `test.url` system property for mock endpoint

## Critical Quirks

* The `backend/README.md` says Maven — **ignore it**. This is Gradle only.
* `kotlin-js-store/` directory is auto-generated by Kotlin/JS webpack plugin; do not commit changes there.
* `.gradle/` is tracked in git in this repo but is build cache — safe to ignore.
